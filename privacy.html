Privacy Policy

Your code stays yours.

Poly-Glot AI is built on a simple principle: your code and your API keys never touch our servers.

Last updated: April 28, 2026

TL;DR — The short version
  • 🔑Your API key is stored locally in your browser or OS keychain — never sent to Poly-Glot.
  • 📡Your code goes directly from your device to your chosen AI provider. Poly-Glot has no backend that ever sees it.
  • 🚫We collect zero telemetry, zero usage data, and zero personal information.
  • 🏠You own your data. Always.
01

Who we are

Poly-Glot AI is an AI-powered code documentation tool available at poly-glot.ai, as a VS Code extension, CLI, MCP server, GitHub App, and Chrome extension. It is developed and operated by Harold Moses.

Contact: hello@poly-glot.ai

02

What data we collect

We collect no personal data. Specifically, Poly-Glot AI does not collect, store, or transmit:

  • Your name, email address, or any personally identifiable information
  • Your source code or any code you paste into the tool
  • Your API keys (OpenAI, Anthropic, Google, or any other provider)
  • Your IP address or device identifiers
  • Usage statistics, analytics, or telemetry of any kind
The only data stored is your settings (provider preference, model choice) and API key — saved locally in your browser's localStorage or your OS keychain (VS Code extension). This data never leaves your device.
03

How the AI generation works

When you generate code comments, Poly-Glot sends your code directly from your browser (or editor) to your chosen AI provider using your own API key. The request path is:

Your device → Your AI provider (OpenAI / Anthropic / Google / etc.) → Your device

Poly-Glot AI has no server or backend in this flow. We never see your code, your API key, or the AI's response.

04

Cookies & local storage

Poly-Glot AI uses browser localStorage to save your preferences (selected provider, model, language, comment style) and your API key on your local device only. We do not use tracking cookies, advertising cookies, or any third-party cookies.

Standard web analytics (Google Analytics 4) is used on the poly-glot.ai website to understand aggregate page traffic. This collects anonymized data such as page views and session counts — no personally identifiable information, no code, no API keys.

06

GitHub App

The Poly-Glot AI GitHub App fetches changed files from pull requests to generate documentation suggestions. This code is:

  • Fetched temporarily at the time of the PR event
  • Sent directly to your configured AI provider using your API key
  • Never stored, logged, or retained by Poly-Glot

The GitHub App requests only the minimum necessary permissions: pull_requests: write, checks: write, contents: read, metadata: read.

07

macOS App — Poly-Glot Prompt Studio

The Poly-Glot Prompt Studio macOS app (App Store ID 6762222589) has the following data practices, separate from the web product:

Anonymous Analytics (optional — requires explicit consent)

  • What is collected: Anonymous usage events such as "app_opened", "template_viewed", "paywall_shown". A randomly generated UUID is stored on your device as a pseudonymous identifier. This UUID is not linked to your name, email address, or Apple ID.
  • Who receives it: PostHog, Inc., operated at us.i.posthog.com (United States). PostHog is SOC 2 Type II certified and provides equivalent data protection standards.
  • Your control: On first launch, the app displays a mandatory consent sheet disclosing exactly what data is sent and who receives it. You may choose "Allow Anonymous Analytics" or "Essential Only — No Analytics". If you choose "Essential Only", no data is ever sent to PostHog. This choice is permanent and stored locally on your device.
  • What is never collected: Your prompt text, template content, API keys, name, email, Apple ID, payment information, or location. All content you create in the app stays entirely on your Mac.

Voice Input (optional — requires macOS system permission)

  • What is collected: Audio is captured only while you actively press "Start Recording" in the Voice Input screen.
  • Who processes it: Apple, Inc., via the macOS Speech Recognition framework. On macOS 13 and later, an on-device neural model is used by default. In some cases, Apple may process a server-side fallback request, as disclosed in the macOS system permission prompt shown by the operating system.
  • Poly-Glot never receives, stores, or transmits any audio or voice transcripts. The transcribed text appears locally in the app and is never sent to our servers.
  • Your control: Voice Input requires an explicit macOS system permission (microphone + speech recognition). You can revoke these at any time in System Settings → Privacy & Security.

In-App Purchases

  • Subscription billing is handled entirely by Apple via StoreKit 2. Poly-Glot never receives or stores your payment details. Purchases are managed through your Apple ID in System Settings → Apple ID → Subscriptions.
09

Data retention

Because we collect no personal data, there is nothing to retain or delete. Your locally stored settings and API key can be cleared at any time by clearing your browser's localStorage or removing the VS Code extension.

If you have a Pro subscription, your email address is retained by our billing provider (Lemon Squeezy) for subscription management. You can request deletion by emailing hello@poly-glot.ai.

Privacy Policy

Prompt Studio Privacy

Your prompts are yours. This page explains how Prompt Studio handles voice audio, template data, API keys, and MCP session tokens.

Last updated: April 6, 2026 · Effective: April 6, 2026

✅ TL;DR — The short version
  • 🎙️Voice audio is never recorded or stored. Transcription happens live in your browser using the Web Speech API — audio never leaves your device.
  • 📝Your templates live in your browser. All prompt templates, variables, and version history are stored in your browser's localStorage — not on our servers.
  • 🔑We never see your API keys. AI provider keys you enter go directly from your browser to your chosen AI provider. Prompt Studio does not proxy or log them.
  • 🔗MCP session tokens are short-lived. Pro/Team tokens are generated on login and scoped to your account only. They expire automatically.
  • 📤Shared prompts are opt-in. Nothing is shared publicly unless you explicitly click "Share via URL". You control visibility.
🎙️
Voice Input — Zero Audio Retention

Voice input uses your browser's built-in Web Speech API (the same technology used by browser address bars and accessibility tools). Audio is processed entirely on your device — no audio stream, recording, or transcript is ever sent to Poly-Glot servers. Once you stop speaking, the transcript text may be used to populate a prompt field — that's it.

02

🎙️ Voice Input

Prompt Studio offers an optional voice input feature that lets you dictate prompt text hands-free in 40+ languages.

How it works technically: Voice input uses the browser-native SpeechRecognition / webkitSpeechRecognition Web API. When you press the mic button, your browser (not Prompt Studio's servers) activates your microphone and performs speech-to-text processing — typically on-device or via the browser vendor's own speech service (e.g. Google Chrome's speech service for Chrome users). Poly-Glot never receives, stores, or has access to your audio.

What we collect from voice input: nothing.

  • Never Audio recordings of your speech
  • Never Voice transcripts stored on our servers
  • Never Language or accent data
  • Never Microphone access outside of active recording sessions
  • Local only The final transcript text is placed into the prompt editor in your browser — it is stored in localStorage with your template if you save it

Browser vendor note: Chrome, Edge, and Safari each route speech recognition differently. Chrome sends audio to Google's speech service to produce the transcript. This is governed by Google's Privacy Policy, not Poly-Glot's. If you prefer fully on-device processing, use a browser with local speech recognition support or disable voice input and type manually.

Microphone permission: Your browser will ask for microphone access the first time you use voice input. You can revoke this at any time in your browser's site settings. Revoking mic access does not affect any other Prompt Studio feature.

📝 Prompt Templates & Version History

All prompt templates you create, edit, or save in Prompt Studio are stored in your browser's localStorage — on your device, in your browser profile.

  • Local only Template name, body, variable definitions
  • Local only Version history (up to 3 versions on Free, unlimited on Pro/Team)
  • Local only Selected AI provider, model, and language preferences

This means your templates are private by default — they exist only in the browser you created them in. If you clear your browser data or use a different browser/device, your locally-saved templates will not be present. We recommend using the export/copy feature to back up important templates.

Shared templates (opt-in only): If you click "Share via URL" (Pro feature), a copy of the template body is stored on Poly-Glot servers and assigned a short URL. The shared copy contains only the template text and variable names — no account info, no API keys, no version history. You can delete a shared URL at any time from your Prompt Studio dashboard.

🔑 API Keys

Prompt Studio supports testing prompts directly against AI providers (OpenAI, Anthropic, Google Gemini, and others) using API keys you supply.

Your API keys are never sent to Poly-Glot. When you enter an API key in Prompt Studio, it is stored in your browser's localStorage and sent directly from your browser to the AI provider's API endpoint when you run a test. The key does not pass through any Poly-Glot server or proxy.
  • Local only API keys stored in browser localStorage — never transmitted to us
  • Never API keys logged, proxied, or cached by Poly-Glot
  • Your responsibility Rotating or revoking keys if you believe they have been compromised

Best practice: Use scoped API keys with usage limits set in your AI provider dashboard. Never paste production keys into browser tools.

🔗 MCP Session Tokens

The Poly-Glot Prompt Studio MCP (Model Context Protocol) server allows Pro and Team subscribers to use Prompt Studio tools directly from any MCP-compatible AI client (e.g. Claude Desktop, Cursor, VS Code Copilot).

To authenticate, a session token (PG_PROMPT_SESSION_TOKEN) is issued to your account after login. Here's how it works:

  • Tokens are tied to your account and cannot be transferred
  • Tokens expire after 30 days of inactivity and can be manually revoked from your dashboard at any time
  • Each MCP API call logs: timestamp, tool name called, and quota consumed — not the prompt content
  • Prompt content sent through MCP tool calls is forwarded to your configured AI provider directly — it is not stored by Poly-Glot
  • Token usage is tracked to enforce plan limits (Free: blocked · Pro: 200 calls/mo · Team: 1,000/mo)
Treat your MCP session token like a password. Anyone with your token can make API calls against your quota. If you suspect a token has been compromised, revoke it immediately from your Prompt Studio dashboard and generate a new one.

👤 Account & Authentication

Prompt Studio uses the same authentication system as poly-glot.ai, powered by Cloudflare Workers. When you sign up or log in:

  • Your email address is stored in Cloudflare KV (encrypted at rest) to manage your account and subscription tier
  • A session token (JWT) is issued and stored in your browser's localStorage
  • No passwords are stored — authentication uses a magic link / OAuth flow
  • Your plan tier (Free / Pro / Team) is stored server-side to enforce feature gates

To delete your account and all associated data, email support@poly-glot.ai with the subject "Delete my Prompt Studio account". We will process deletion within 5 business days.

07

💳 Subscriptions & Billing

Pro and Team subscriptions are handled by Stripe. Poly-Glot never sees or stores your full payment card details — all billing data is handled entirely by Stripe and governed by Stripe's Privacy Policy ↗.

  • Your email address is shared with Stripe to create a billing customer record
  • Your subscription status (active / cancelled) is stored in Cloudflare KV to gate Pro/Team features
  • No credit card numbers, CVVs, or billing addresses are ever stored by Poly-Glot

🌍 GDPR & CCPA

If you are located in the European Union or California, you have additional rights regarding your personal data:

  • Right to access — request a copy of the personal data we hold about you
  • Right to erasure — request deletion of your account and associated server-side data
  • Right to portability — request your data in a machine-readable format
  • Right to object — object to processing of your personal data
  • Right to correct — request correction of inaccurate data

Because most Prompt Studio data lives in your own browser (localStorage), you already have direct control over the majority of your data. For server-side data (account record, shared templates, MCP logs), contact us at support@poly-glot.ai.

No voice data GDPR concerns: Voice audio is processed entirely by your browser and is never transmitted to or stored by Poly-Glot. There is no voice data subject to GDPR access or deletion requests on our end.
